# 系统总体架构设计

## 1. 架构概览

系统采用三层架构：

- 表现层：Unity3D 虚拟校园场景与交互界面
- 服务层：FastAPI 提供路线规划、AIGC 讲解、问答和语音接口
- 数据层：校园节点图数据、导览日志、用户画像

## 2. 核心模块

### 2.1 路线规划模块

- 输入：起点、终点、偏好（避拥挤、室内优先）
- 方法：加权 Dijkstra
- 输出：节点路径、分段距离、总时长

### 2.2 AIGC 讲解模块

- 输入：用户角色、风格、路线节点
- 方法：模板兜底 + 大模型生成
- 输出：个性化讲解文本

### 2.3 智能问答模块

- 输入：自然语言问题 + 当前景点上下文
- 方法：上下文增强生成
- 输出：简洁问答结果

### 2.4 语音模块

- 输入：讲解文本
- 方法：TTS（当前为 mock，支持扩展真实云服务）
- 输出：可播放音频 URL

## 3. 关键数据结构

- CampusNode: 节点 ID、名称、类别、简介、室内属性、拥挤度
- CampusEdge: 起终节点、距离、室内属性、拥挤度

## 4. 交互流程

1) Unity 发送路线规划请求
2) 后端返回最优路径
3) Unity 发起讲解生成请求
4) 后端调用 AIGC（或模板兜底）返回讲解词
5) Unity 展示文本并可请求语音播放

## 5. 性能与可扩展性

- 路径图可扩展至校园全图并按区域分片加载
- 支持缓存热门路线和讲解结果
- 支持多模型切换与私有化部署
